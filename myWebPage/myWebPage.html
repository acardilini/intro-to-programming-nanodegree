Understanding the Web

In it's simplest form the World Wide Web (WWW) is a collection of hyperlinked HTML documents. There are several components of the web which are required for it to function including, you and your hardware and a browser, the internet and severs. The browser sends HTTP requests to servers via the internet which then send back files in response, these files are interpretted and displayed by your browser.

HTML

HTML, which stands for HyperText Markup Language, documentsare are the heart of the web and tell browsers what pages should look like. These documents are written in a text language that browsers can interpret to create webpages, links to other pages, embed media and make it possible to create the web as we know it.

The Markup Language

The markup language is made up of tags which define the proporties of an element. The majority of tags contain an opening and closing tag, with the content between these taking on the properties defined by the tag. The properties of a tag are determined by it's name, for instance, to create text that is bold, it is wrapped by tags which specify that the text should be bold: e.g. <b>CONTENTS</b>. Where the tag name = b and defines that the contents should be bold. You will notice that the closing tag has a forward slash '/' before the tag name, this is to distinguish between opening and closing tags. Attributes

Tags can also contain attributes, with the value of the attribute defin or additional properties of the content of the tag. This is best highlighted using the tag for creating hypelinks because it always requires a specific attribute. The tag is <a href="LINK">>CONTENTS</a>. As you can see this tag includes an attribute, href=, in the opening tag. The href attribute specifies the URL of the page that the link will go to, with the contents being set up as a link. There is a good explaination this tag at W3Schools. Another example of a tag which reqires attributes is the <img> tag which makes it possible to include images on a webpage. This tag requires two standard attributes src=, which indicates the location of the image to be used, and alt=, which specifies text to display if the src link is broken. The image tag is another example of a tag without a closing tag because it doesn't require content.

Container Tags

Container tags are simply tags which do nothing but contain the content within them, and which can be used to add style to content. Two container tags include,

  • <span>, an inline tag; and
  • <div> a block tag.
All tags have a predefined position type, which specifies whether they are inline or block. This determines how the content within the tag behaves in relationship to the elements around it, e.g. other tags. Inline elements will sit next to one another on the same line, whereas, block elements will cause a line break and sit below the preceeding element. Inline elements include, emphasis and bold, which as you can see sit side by side with other elements. While, paragraph and headingX, are block elements which sit on their own lines.

It is also important to note that any whitespace with HTML code is turned into a single space. This includes multiple spaces and enters which will all be turned into a single whitespace when rendered.

Stage 1: 1.2 Creating a Structured Document

This lesson focuses on developing an understanding of how a HTML document is structured and how web browsers interpret that structure. It also goes through the purpose of HTML, CSS and DOM.

Why start learning programming with front-end web development?
Understanding front end web dev is one of the few areas of programming that is visual, which allows new students to immediately see the effects of their programming. This is a good feedback mechanism for beginner programmers who are just starting to learn. It's also an easy way to share what you are creating, which allows you to show off your skills to others easily and get feedback.

Concepts of Webpage Structure

Having a conceptual understanding of how the different components of a webpage work will help translate visual designs into webpages. HTML provides the structure of a page, whereas CSS defines the style of the house, such as the colouring and decoration. Browsers build a branching tree-like structure of the page, which represents the association between different elements.

Inspecting the Web

Many web browsers have development tools that allow you to inspect the structure and underlying code that defines a webpage. You can use google chromes development tools by right clicking anywhere on a webpage and selecting 'inspect'. This shows both HTML and CSS style, and can be directly interacted with to see how changes effect the webpage.

A Webpage Trinity (HTML-CSS-DOM)

HTML and CSS are both language of the web with a specific syntax and sets of rules. The basic word in the HTML language is 'tag', which browsers turn into elements that form a tree (or DOM). The Document Object Model (DOM) is a standard way of representing and interacting with HTML tags, and the tree that is seen in a browsers dev tool is actually a DOM. Each HTML tag creates an element in the DOM which the browser use to create the webpage. The HTML document identifies what CSS stylesheet that should be used to style the webpage.

With a good understand of both HTML and CSS languages it is possible to look at a page and see how it can be broken down into it's different elements, or how a page might be built.

Boxes, Boxes Everywhere!

Every element on a webpage is a box, even circles are boxes! CSS can be used to style elements, such as coloured boxes, and make the look like circles, e.g. 'border-radius'. However, the element is still fundamentally a box, its just that it's style has been changed.

Thinking about the elements of a website in terms of boxes makes it simple to organise and rearrange webpages. It allows you to think about a webpage in terms of a grid, and identify where each element will be placed within that grid.

When you are given a mock design, it can helpful to boxify the design by drawing boxes around the different elements you have identified.

Boxes are defined by <div> tags. When creating a new webpage, start by setting up all of the boxes/div tags you need. To be able to apply style to your divs, e.g. size and location, it is helpful to give each element a class attribute which distinguishes it's purpose. CSS can then be used to style each of the different divs depending on its class. It's important to make each class attribute meaningful to the div that it identifies so that you recognise what it is at a later date.
