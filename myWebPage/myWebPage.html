1.1 The Basics of the Web and HTML

Understanding the Web

In it's simplest form the World Wide Web (WWW) is a collection of hyperlinked HTML documents. There are several components of the web which are required for it to function including, you and your hardware and a browser, the internet and severs. The browser sends HTTP requests to servers via the internet which then send back files in response, these files are interpretted and displayed by your browser.

HTML

HTML, which stands for HyperText Markup Language, documentsare are the heart of the web and tell browsers what pages should look like. These documents are written in a text language that browsers can interpret to create webpages, links to other pages, embed media and make it possible to create the web as we know it.

The Markup Language

The markup language is made up of tags which define the proporties of an element. The majority of tags contain an opening and closing tag, with the content between these taking on the properties defined by the tag. The properties of a tag are determined by it's name, for instance, to create text that is bold, it is wrapped by tags which specify that the text should be bold: e.g. <b>CONTENTS</b>. Where the tag name = b and defines that the contents should be bold. You will notice that the closing tag has a forward slash '/' before the tag name, this is to distinguish between opening and closing tags. There are also a small number of tags which don't have a closing tag because they don't contain content. An example of this is the <br> tag which specifies when a line break should occur.

So if a big occurs it will be a mistake that you have accidently written (or not written) into the code.

Attributes

Tags can also contain attributes, with the value of the attribute defin or additional properties of the content of the tag. This is best highlighted using the tag for creating hypelinks because it always requires a specific attribute. The tag is <a href="LINK">CONTENTS</a>. As you can see this tag includes an attribute, href=, in the opening tag. The href attribute specifies the URL of the page that the link will go to, with the contents being set up as a link. There is a good explaination this tag at W3Schools.

Another example of a tag which reqires attributes is the <img> tag which makes it possible to include images on a webpage. This tag requires two standard attributes src=, which indicates the location of the image to be used, and alt=, which specifies text to display if the src link is broken. The image tag is another example of a tag without a closing tag because it doesn't require content.

Container Tags

Container tags are simply tags which do nothing but contain the content within them, and which can be used to add style to content. Two container tags include,

  • <span>, an inline tag; and
  • <div> a block tag.

All tags have a predefined position type, which specifies whether they are inline or block. This determines how the content within the tag behaves in relationship to the elements around it, e.g. other tags. Inline elements will sit next to one another on the same line, whereas, block elements will cause a line break and sit below the preceeding element. Inline elements include, emphasis and bold, which as you can see sit side by side with other elements. While, paragraph and headingX, are block elements which sit on their own lines.

It is also important to note that any whitespace with HTML code is turned into a single space. This includes multiple spaces and enters which will all be turned into a single whitespace when rendered.

HTML Document Setup

To create a HTML document it must contain a few vital elements which include:

  • <!DOCTYPE HTML> - defines what kind of html the document is.
  • <HTML> - Surrounds the entrie document
  • <head> - Contains metadata and other scripts like, CSS and Javascrip
  • <title> - The title of the page or the URL
  • <meta charset="utf-8"> - Specific the character encoding for the HTML doc
  • <body> - Everything you actually see on the webpage.

1.2 Creating a Structured Document

This lesson focuses on developing an understanding of how a HTML document is structured and how web browsers interpret that structure. It also goes through the purpose of HTML, CSS and DOM.

Why start learning programming with front-end web development?
Understanding front end web dev is one of the few areas of programming that is visual, which allows new students to immediately see the effects of their programming. This is a good feedback mechanism for beginner programmers who are just starting to learn. It's also an easy way to share what you are creating, which allows you to show off your skills to others easily and get feedback.

Concepts of Webpage Structure

Having a conceptual understanding of how the different components of a webpage work will help translate visual designs into webpages. HTML provides the structure of a page, whereas CSS defines the style of the house, such as the colouring and decoration. Browsers build a branching tree-like structure of the page, which represents the association between different elements.

Inspecting the Web

Many web browsers have development tools that allow you to inspect the structure and underlying code that defines a webpage. You can use google chromes development tools by right clicking anywhere on a webpage and selecting 'inspect'. This shows both HTML and CSS style, and can be directly interacted with to see how changes effect the webpage.

A Webpage Trinity (HTML-CSS-DOM)

HTML and CSS are both language of the web with a specific syntax and sets of rules. The basic word in the HTML language is 'tag', which browsers turn into elements that form a tree (or DOM). The Document Object Model (DOM) is a standard way of representing and interacting with HTML tags, and the tree that is seen in a browsers dev tool is actually a DOM. Each HTML tag creates an element in the DOM which the browser use to create the webpage. The HTML document identifies what CSS stylesheet that should be used to style the webpage.

With a good understand of both HTML and CSS languages it is possible to look at a page and see how it can be broken down into it's different elements, or how a page might be built.

The Web is Full of Boxes!

Every element on a webpage is a box, even circles are boxes! CSS can be used to style elements, such as coloured boxes, and make the look like circles, e.g. 'border-radius'. However, the element is still fundamentally a box, its just that it's style has been changed.

Thinking about the elements of a website in terms of boxes makes it simple to organise and rearrange webpages. It allows you to think about a webpage in terms of a grid, and identify where each element will be placed within that grid.

When you are given a mock design, it can helpful to boxify the design by drawing boxes around the different elements you have identified.

Boxes are defined by <div> tags. When creating a new webpage, start by setting up all of the boxes/div tags you need. To be able to apply style to your divs, e.g. size and location, it is helpful to give each element a class attribute which distinguishes it's purpose. CSS can then be used to style each of the different divs depending on its class. It's important to make each class attribute meaningful to the div that it identifies so that you recognise what it is at a later date.

1.3 Adding CSS Style and HTML Structure

In this lesson we learn how to use CSS code to add style to a webpage.

The Basics of Style

There are several ways to add style to a HTML document including directly within each tag, between <style> tags in the <head> of the head of the HTML doc, or by using a CSS stylesheet.

Using a CSS reduces the amount of repitive coding that needs to be done because it can be linked to many pages at once, and one change could lead to changes in many documents. For example, if you style was between <style> across multiple webpages you would need to change every webpage seperately, however with CSS you only have to change it once in a single CSS file that is linked to all of the pages!

Your HTML document is linked to a CSS document by including a <link rel="stylesheet" type="text/css" href="LINK"> tag, between the <head> tags of your HTML.

Understanding CSS

CSS stands for Cascading Style Sheets. Multiple CSS can be linked to a single HTML document, with the most specific rules applying the style to the webpage. That is, there can be multple rules which govern how an element should be styled but it is the most specific rule that is applied. An Inheritance mechanisms helps determine what styling is applied to elements of a HTML document, with descendants of an element, i.e. tags nested in other tags, inheriting the style of their ancestor elements. The inheritance mechanism relies on the tree-like structure of the DOM to determine styling for descendants. The style applied directly to a descendant overrides the style inherited from an ancestor, i.e. the most specific rule is applied.

CSS starts with a selector, which identifies what elements to apply the style to. Selectors can be tag names, and will apply to all elements of that tag, or can be more specific and target elements of a specific class. CSS syntax follows this simple structure: 

          SELECTOR {
            ATTRIBUTE: VALUE;
          }

CSS-Tricks (which is a great website and youtube channel), provides a good introduction to CSS-Selectors which is well worth checking out. There are also LOTS of different attributes that can be specified to style HTML elements. Mozilla has a CSS reference page that provides information about lots of these attributes and the values that can be applied to them. It is important to note that there are often several ways of defining the value of an attribute.

Expert-Tips

These tips were taken from a udacity video chat that was focussed on discussion good practice for formatting your HTML and CSS code. This topic was discussed to highlight the importance of good code writing format, which helps you write more efficient code, helps you understand the code more effeciently and helps foster good communication with other coders.

  • HTML
    • Clean, consistent and logical indentation is very important!

  • CSS

    • Be consistent with the placement of your {} and spacing.

    • Use a 'TODO:' starter to make a comment about work you need to do in the future.
    • Add comments that descripte how you are doing what you're doing or why you are doing what you are doing. Don't describe what you are doing because the code does this for you!

The Box Model

The box model which describes the components that every element of a webpage consists of. These components include a margin, border, padding and content. Each of the components plays a specific role in providing style to an elements:

  • Content is the image or text that appears on a webpage, it sits at the centre of the elements box. The other parts of the box 'protect' the content so that other webpage elements don't sit to close or obscure it.
  • Padding clears the area around the content and is effected by the background color of the box.
  • Border goes around the content and padding of the box.
  • Margin clears an area around the boarder and is completely transparent.

By default the size of an element is equal to the sum of the box component pixels. This can make it hard to know how much space the box will take up on the screen especially if you are changing elements often. A simpler model, called box-sizing has been developed with this in mind and calculates the element size inlcuding both the border and padding. It means you can set the overall box size and tweaking the padding and border will not change it. Margin is not included in this size. There is a drawback to using box-sizing. It's relatively new and special prefixes must be included for some of the older browsers to use it: 

          * {
            -webkit-box-sizing: border-box;
            -moz-box-sizing: border-box;
            -ms-box-sizing: border-box;
            box-sizing: border-box;
          }

From Design to Webpage

The Code, Test and Refine process is a good way to work towards creating a design you have been given. The following steps describe a basic form form of this process:

  1. Boxify the design.
  2. Look for repeated style and symatic elements, i.e. figure out what tags are required.
  3. Write the HTML.
  4. Apply style, from the biggest to smallest elements.
  5. Fix things. Look at the site in different browsers and repeat steps 3 and 4 until the webpage looks like the design mock.

Verification

It is important that you validate your code to check that it will work on all of the different browsers available. Online tools help check HTML and CSS. These methods provide validation icons that can be added to your website to show that it meets appropriate standards.
