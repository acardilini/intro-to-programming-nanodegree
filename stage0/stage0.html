Stage0: Getting Started with HTML

Understanding the Web

In it's simplest form the World Wide Web (WWW) is a collection of hyperlinked HTML documents. There are several components of the web which are required for it to function including, you and your hardware and a browser, the internet and severs. The browser sends HTTP requests to servers via the internet which then send back files in response, these files are interpretted and displayed by your browser.

HTML

HTML, which stands for HyperText Markup Language, documentsare are the heart of the web and tell browsers what pages should look like. These documents are written in a text language that browsers can interpret to create webpages, links to other pages, embed media and make it possible to create the web as we know it.

The Markup Language

The markup language is made up of tags which define the proporties of an element. The majority of tags contain an opening and closing tag, with the content between these taking on the properties defined by the tag. The properties of a tag are determined by it's name, for instance, to create text that is bold, it is wrapped by tags which specify that the text should be bold: e.g. <b>CONTENTS</b>. Where the tag name = b and defines that the contents should be bold. You will notice that the closing tag has a forward slash '/' before the tag name, this is to distinguish between opening and closing tags. Attributes

Tags can also contain attributes, with the value of the attribute defin or additional properties of the content of the tag. This is best highlighted using the tag for creating hypelinks because it always requires a specific attribute. The tag is <a href="LINK">CONTENTSa>. As you can see this tag includes an attribute, href=, in the opening tag. The href attribute specifies the URL of the page that the link will go to, with the contents being set up as a link. There is a good explaination this tag at W3Schools. Another example of a tag which reqires attributes is the <img> tag which makes it possible to include images on a webpage. This tag requires two standard attributes src=, which indicates the location of the image to be used, and alt=, which specifies text to display if the src link is broken. The image tag is another example of a tag without a closing tag because it doesn't require content.

Container Tags

Container tags are simply tags which do nothing but contain the content within them, and which can be used to add style to content. Two container tags include,

  • <span>, an inline tag; and
  • <div>
    • a block tag.
All tags have a predefined position type, which specifies whether they are inline or block. This determines how the content within the tag behaves in relationship to the elements around it, e.g. other tags. Inline elements will sit next to one another on the same line, whereas, block elements will cause a line break and sit below the preceeding element. Inline elements include, emphasis and bold, which as you can see sit side by side with other elements. While, paragraph and headingX, are block elements which sit on their own lines.

It is also important to note that any whitespace with HTML code is turned into a single space. This includes multiple spaces and enters which will all be turned into a single whitespace when rendered.

HTML Document Setup

To create a HTML document it must contain a few vital elements which include:

  • <!DOCTYPE HTML> - defines what kind of html the document is.
  • <HTML> - Surrounds the entrie document
  • <head> - Contains metadata and other scripts like, CSS and Javascrip
  • <title> - The title of the page or the URL
  • <body> - Everything you actually see on the webpage.
